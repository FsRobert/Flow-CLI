概述

关于

front-flow-cli是front-flow前端工作流的配套命令行工具，实现自动初始化、组件化开发、预处理、静态资源优化、图片压缩等前端自动化需求。

典型的front-flow模板项目结构如下，其中下划线(_)开头的目录为编译资源，编译后不需要部署到服务器

             
  |--_component/                //UI组件
  |-- _src/                     //源码目录
  |    |-- css/
  |    |-- font/
  |    |-- img/
  |    |-- include/
  |    |-- js/
  |    |-- lib/
  |    |-- index.html
  |    |-- seajs.config.js
  |    `-- favicon.ico 
  |-- modules/                  //seajs-modules
  `-- config.json               //编译配置

安装

1. 安装nodejs(>=6.5.0)

2. npm全局安装：npm install front-flow-cli -g

3. 安装完成，查看版本号：flow -V

命令

1. flow init在当前目录初始化一个front-flow-template项目

2. flow build编译当前项目

3. flow run编译当前项目并启动开发服务，实时编译并刷新浏览器

配置

编译相关的所有配置均在config.json中设置：
projectDir 项目源码目录，默认"_src"
distDir 编译产出目录，默认"dist"
port 开发服务端口号，默认3000
homePage 开发服务起始页面，默认"index.html"
serverRoot 站点部署路径，默认""，即根目录
reloadDelay 开发服务刷新延时，默认0，单位ms
rootHolder 源码中引用站点部署路径的变量名
projectHolder 源码中引用项目源码目录的变量名
distHolder 源码中引用产出目录的变量名
UUIDHolder 源码中生成一个随机6位数字字母组合的变量名，仅用于组件中html和script成对使用
checkUpdate 是否自动检测新版本，默认true
compress 编译时是否压缩代码，默认true

编译过程

UI

_component/用于存放UI组件，_src/css/config.less是less配置文件，_src/css/responsive.less、_src/css/resp-midd.less、_src/css/resp-smal.less是响应式样式文件，这些文件可以按需在_src/css/style.less中引用，最终编译到style.css中，不会单独输出到产出文件，他们的修改会触发_src/css/style.less的重新编译；

项目里的其他css/less文件将按原相对路径输出到产出文件夹。
